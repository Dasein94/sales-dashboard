# Sales Dashboard

A minimal end-to-end sales reporting system built across two sessions.

Python reads a CSV file, computes revenue summaries with pandas, and exports
the results as JSON files. A static HTML/JS dashboard reads those JSON files
and renders charts using Chart.js. When you're ready to publish, a Python
script uploads everything to your web host via SFTP — no backend required.

---

## Prerequisites

- Python 3.13 or newer
- [uv](https://docs.astral.sh/uv/) (recommended package manager) — or pip
- A text editor
- *(Session 2)* SFTP credentials for your web host
- *(Final Step)* Gemini API key — get one free at [Google AI Studio](https://aistudio.google.com/app/apikey)

## Session 1: Data Pipeline

### Step 1 — Set up dependencies

Install the project dependencies into a virtual environment:

```bash
uv sync
```

This reads `pyproject.toml` and installs pandas, openpyxl, and python-dotenv
into a local `.venv` folder. You only need to do this once.

> **With pip instead of uv:**
> ```bash
> python -m venv .venv
> source .venv/bin/activate      # Windows: .venv\Scripts\activate
> pip install pandas openpyxl python-dotenv
> ```

---

### Step 2 — Review the sample sales data

Open `data/sales.csv`. It contains 50 rows of sales transactions with these columns:

| Column | Description |
|---|---|
| `date` | Transaction date (YYYY-MM-DD) |
| `region` | Sales region: North, South, East, or West |
| `product` | Product name: Laptop, Monitor, Keyboard, or Mouse |
| `units_sold` | Number of units sold in this transaction |
| `unit_price` | Price per unit (consistent per product) |
| `revenue` | `units_sold × unit_price` (pre-computed) |

Take a moment to scroll through the file and verify a few rows by hand.

---

### Step 3 — Walk through analyze.py

Open `analyze.py`. It contains six functions — read through each one:

| Function | What it does |
|---|---|
| `load_data()` | Reads the CSV and parses the date column |
| `compute_summary()` | Totals revenue and units; finds top product and region |
| `compute_monthly_revenue()` | Groups revenue by calendar month |
| `compute_revenue_by_product()` | Groups revenue by product |
| `compute_revenue_by_region()` | Groups revenue by region |
| `write_json(data, filename)` | Saves a Python dict or list as a JSON file |

Key pandas concepts used:
- `pd.read_csv(..., parse_dates=["date"])` — load and parse dates in one step
- `df.groupby("column")["value"].sum()` — split-apply-combine pattern
- `.idxmax()` — returns the label (name) of the highest value
- `.dt.to_period("M")` — reduce a date to just the year-month
- `.to_dict(orient="records")` — convert a DataFrame to a list of dicts

---

### Step 4 — Run and inspect the output

```bash
python analyze.py
```

Expected output:

## Session 2: Dashboard & Deployment

### Step 5 — Build the HTML layout

Create `dashboard/index.html` with four sections:
- A row of KPI cards (total revenue, total units, top product, top region)
- A line chart canvas for monthly revenue
- A bar chart canvas for revenue by product
- A pie chart canvas for revenue by region

Load Chart.js from a CDN at the bottom of the page — no npm needed.

---

### Step 6 — Write app.js (fetch + Chart.js)

Create `dashboard/app.js`:
- Use `fetch()` to load the four JSON files
- Wrap all four calls in `Promise.all()` so they load in parallel
- Pass the data to Chart.js to render each chart
- Add a `try/catch` so errors show a friendly message instead of a blank page

---

### Step 7 — Write upload.py (SFTP)

Create `upload.py`:
- Read SFTP credentials from `.env` using `python-dotenv`
- Connect with `paramiko.SSHClient` (Python standard library — no extra install needed)
- Upload everything in `output/` and `dashboard/` to the remote server

---

### Step 8 — Wire main.py

Update `main.py` to import and call `analyze.main()` then `upload.main()`
in sequence — one command runs the full pipeline.

---

### Step 9 — Deploy and verify

1. Copy the credentials template and fill it in:
   ```bash
   cp .env.example .env
   ```
   Open `.env` and enter your SFTP host, username, password, and remote directory.

2. Run the full pipeline:
   ```bash
   python main.py
   ```

3. Visit your domain in a browser and confirm the dashboard loads with live data.

> **Testing locally before deploying:**
> Because `fetch()` is blocked when opening files directly (`file://` URLs),
> always test with a local server:
> ```bash
> python -m http.server 8000
> ```
> Then open `http://localhost:8000/dashboard/` in your browser.

## Final Step: AI-generated Insights with Gemini

### Step 10 — Add the Gemini API key

Get a free key at [Google AI Studio](https://aistudio.google.com/app/apikey) and add it to `.env`:

```
GEMINI_API_KEY=your_gemini_api_key
```

---

### Step 11 — Walk through compute_insights()

Open `analyze.py` and find `compute_insights()`. It builds a compact text summary
from the already-computed aggregates and sends it to the Gemini API with a
business-analyst prompt:

| Part | What it does |
|---|---|
| `os.environ.get("GEMINI_API_KEY")` | Reads the API key loaded from `.env` |
| `compute_monthly_revenue()` etc. | Builds a compact summary to send as context |
| `genai.GenerativeModel("gemini-2.5-flash")` | Selects the Gemini model |
| `model.generate_content(prompt)` | Sends the prompt and returns the AI response |

The response is 2–3 sentences of plain-English insight, saved to `output/insights.json`.

---

### Step 12 — Review the dashboard changes

`dashboard/index.html` adds an AI Insights section below the KPI cards:
```html
<section class="chart-section">
  <h2>AI Insights</h2>
  <p id="ai-insights">Loading insights...</p>
</section>
```

`dashboard/app.js` loads `insights.json` in the same `Promise.all()` call and injects the text:
```js
const [summary, insights, monthly, byProduct, byRegion] = await Promise.all([
  loadJSON("summary.json"),
  loadJSON("insights.json"),
  loadJSON("monthly_revenue.json"),
  loadJSON("revenue_by_product.json"),
  loadJSON("revenue_by_region.json"),
]);
document.getElementById("ai-insights").textContent = insights.insights;
```

Open `http://localhost:8000/dashboard/` — the AI Insights section should populate
